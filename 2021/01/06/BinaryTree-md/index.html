    Efficiency

    1. A single statement = 1
    2. A function call’s runtime =( sum of runtime of statement in function’s body)

    -3.A loop of N iterterators’ runtime = (N*(loop body’s runtime))

    • Growth rates: Change in runtime as input size “N” changes
    • The higest-order term dominates the overall runtime ( such as Big-Oh of N cubed)

    Vector Effiency

    Member Big-Oh * Comment
    v.add(value) O(1) It dont need to move the whole queue
    v.get(i) or v[i] O(1) It dont need to move the whole queue
    v.insert(i, value) O(N) It need to shift
    v.remove(i) O(N) It need to shift the internal array
    v.set(i，value) or v[i] O(1) It dont need to traverse
    v.size() or v[.isEmpty() O(1) It dont need to loop over or ship
    v.toString or cout< O(N) It need to loop over the internal array

    Set Effiency

    Member Set Hash Set Comment
    s.add(value) O(log N) O(1) Log N is chance geting smaller
    s.clear() O(N) O(N) remove all elements of set
    s.contains(value) O(log N) O(1) return true if the given value is found
    s.remove(value) O(log N) O(1) It need to shift the internal array
    s.isSubsetOf(set) O(N log N) O(N) True if the given set is contained
    s.isEmpty() O(1) O(1) It dont need to loop over or ship
    s.size() O(1) O(1) It dont need to loop over or ship
    s.toString O(N) O(N) It need to loop over the internal array; e.g. "{3, 42, -7, 15}"
    ostr< O(N) O(N) print set to stream

    Binary Tree

    • One where each noe has at most two children

    • Recursive definition: A tree is either: n

      • empty (nullptr), or
      • a root node that contains:
        1. data,
        2. a left subtree,
        3. a right subtree

    • Node: an object containing a data value and left/right children

      • root: topmost node of a tree
      • leaf: a node that has no children
      • branch: any internal node; neither the root nor a leaf
      • parent: a node that refers to this one
      • a noode that this node refers to
      • sibling: a node with a common
      • subtree: the smaller tree of nodes on the left or right of the current node
      • height: length of the longest path from the root to any node
      • level or depth: length of the path from a root to a given node
