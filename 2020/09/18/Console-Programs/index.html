Reference parameters

Purpose

Normally, the function uses the copy of calling value of parameters and change local copy instead of calling argument itself. In order to change the calling argument, value parameter is required to be changed to reference parameters.

Use scenarios

The function needs to return mutiple values to the calling program.

<cmath> library

abs(x) sqrt(x) floor(x) ceil(x)
exp(x) log(x) log10(x) pow(x, y)
cos(theta) sin(theta) tan(theta) atan(x)
atan2(y,x)

Interfaces and implementations

  • Interface:
    • provide information that clients need to use the library but without the detail about how the library works.
    • the name of interface file: end with .h , which is a header file.
  • Implementation:
    • specifies the underlying details
    • export several definition including functions, types and constants
    • each individual definition is called interface entry
    • implementation file: end with .cpp
  1. Export a single function
    • Take defining the error library as an example
  • interface boilerplate: is a patterned set of lines that appears in every interface (#findef, #define and #endif)

Explain

  • The cerr output stream
    • similar with cout, cerr for reporting error.
  • The exit function
    • terminates the execution of the main program immediately
  • The constant EXIT_FAILURE
    • defined in the library and is used to indicate that some kind of failure occurred
  1. Expor data types
    Most of these types will be classes, taking an example of creating an interface that exports one of the enumerated types.
  1. Exporting constant definitions
    const double PI = 3.14159265358979323846;

Principles of interface design

  • Unified
    • Each interface entry exported by libraries fits the purpose of interface
    • The functions within an interface should behave in as consistent a way as possible
  • Simple
    • reduce the complexity of the programming process
    • information hiding: Keeping details confined to the implementation domain
  • Sufficient
    • the efficiency of its implementation: meeting the needs of clients
  • General
    • it must be general enough to solve a wide range of problems and not be limited to one highly specific purpose
  • Scable
    • Changing an interface in such a relatively straightforward process, since no clients already depend on that functionthat existing programs will continue to run without modification is called extending the interface

Exercise

Exercise 01 (7)

Even though clients of the library typically don’t need to understand how functions like sqrt work internally, the implementers of that library have to be able to design an effective algorithm and write the necessary code. If you were asked to implement the sqrt function without using the library version, there are many strategies you could adopt. One of the easiest strategies to understand is successive approximation in which you make a guess at the solution and then refine that guess by choosing new values that move closer to the solution

You can use successive approximation to determine the square root of x by adopting the following strategy:

  1. Begin by guessing that the square root is x / 2. Call that guess g.
  2. The actual square root must lie between g and x / g. At each step in the successive approximation, generate a new guess by averaging g and x / g.
  3. Repeat step 2 until the values g and x / g are as close together as the machine precision allows. In C++, the best way to check for this condition is to test whether the average is equal to either of the values used to generate it. Use this strategy to write your own implementation of the sqrt function.
