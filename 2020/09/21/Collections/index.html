Collections

Vector

  1. Array weakness

    • fixed size, cannot subsequently change
    • Additional valiable to track the numbers of elements in array
    • not support to insert and delete elements

  2. Vectors solve these problem array has

    • specify the base type of a vector
      • Vector
    • declare a vector object
      • e.g. Vector vec
    • operation in vector.h interface
      • add: vec.add(element); vec.insertAt(pos, element)
      • remove: vec.removeAt(pos);
      • select: vec.get(pos); vec[pos];
      • modify: vec.set(pos, value);
      • constructor: Vector (): create an empty vector; Vector (n, value): create a vector with n elements, each of which is initialized to value (default value if value is missing).
      • default value: 0 for numberic types; false for bool; the character whoes ASCII is 0 for char;
      • size(); isEmpty(); clear();
      • v1+v2 : conatenates v1 and v2 and return a new vector containing the combined elements
      • vec += e1, e2,… add elements e1, e2 and so on, to the end of vector.
    • pass a vector object as a parameter
      • call-by-reference
    • a predefined size for vector
    • two-dimentional structure
      Vector< Vector<int> > sudoku(9, Vector<int>(9));
      • The base type of the inner vector is int, and the base type of the outer vector is Vector.

  3. Grid class in the Stanford libraries

    • For two-dimentional sturcture of vector, called grid
    • Constructor: Grid (): must specify the grid dimensions by calling resize; Grid (rows, cols)
    • Methods: numRows(); numCols(); bool inBound(row, col); get(row, col); set (row, col, value); resize (rows, cols);

LinkedList

# include “linkedlist.h”

  • Class LinkedList provides the same functionality as Vector.
  • Made of nodes, each storing a value and link to ‘next’ node.
  • Difference internal data stucture between vector and linkedlist:
    • linkedlist knows the front node
    • linkedlist contain new element with no shifting

Stacks

  1. LIFO, which stands for “last in, first out.”
  2. Push value: add
  3. Poping the stack: removing the most recent item from a stack
  4. stack.h interface
    Stack () size() isEmpty() push(value)
    pop () peak() clear()
    5. Function call= push; return =pop

Queues

  1. FIFO, “first in, first out,”
  2. enqueue: adds a new element to the end ( tail).
  3. dequeue: removes the element at the beginning of the queue (head).
  4. peak: examine the front element
  5. queue.h interface
    Queue () size() isEmpty() enqueue(value)
    dequeue () peak() clear()

Map

  1. pairs: a key and an associated value
  2. declaring a dictionary variable
    Map<string,string> dictionary;
  3. map.h interface
    Map () size() isEmpty() put(key, value)
    get (key) remove(key) containKey(key) clear()
    toString() values(): a vector cope of all values in map keys(): a vector cope of all keys in map ostr<
  4. Difference between Map and HashMap
    • map: sorted order; pretty fast
    • hashmap: hashtable, unpredictable order; very fast
  5. Loop
    • the range-based for loop returns the keys in the natural order for the key type
      • integers: the keys in ascending numerical order
      • strings: the keys in lexicographic order

Set

  1. A collection of unique values (no duplicates allowed)
  2. set.h interface
    Set () size() isEmpty() add(value)
    remove (value) contains(value) clear() isSubsetOf(set)
    first() union: s1+ s2 intersection: s1* s2 (common) set difference: s1 - s2
    s1+= s2 s1* =s2 (common) s1 -= s2
  3. Lexicon class in the Stanford libraries
    Lexicon () Lexicon (file) size() isEmpty()
    add(word) addWordsFromFile (file) contains(world) containsPrefix(prefix)
    clear()
  4. Difference between Set and HashSet
    • set: sorted order; pretty fast
    • hashset: unpredictable order; very fast
  5. Loop
    • the range-based for loop always returns the elements in the natural order for the value type
